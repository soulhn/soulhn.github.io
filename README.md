# 리빙 포트폴리오 — [soulhn.github.io](https://soulhn.github.io)

교육 PM × AI 빌더의 역량 지도와 빌드 로그. **단일 HTML + 바닐라 JS, 프레임워크 없음.**

## Build 01 리포트 (2026.07)

**문제** — 증거(대시보드·repo·블로그)가 흩어져 있어, 나를 3분 안에 파악할 수 있는 단일 URL이 없었다.

**만든 것 / 버린 것** — 인터랙티브 역량 지도(7축 레이더)와 Build Log를 담은 원페이지를 만들고, 블로그 엔진·자동 발행 파이프라인은 다음 빌드로 미뤘다(스코프 절단).

**AI에게 맡긴 것** — 코드 전량(SVG 레이더 지오메트리·인터랙션 포함), 구조화해둔 커리어 데이터 기반 콘텐츠 초안, 배포 자동화.

**직접 판단한 것** — 콘셉트 전환("이력서가 아니라 작업실"), 라이트 테마 전환(독자 기준), 공개 범위와 표현의 수위, 스코프 절단 — 방향 수정 약 20회.

**검증** — 완료 기준 5개를 먼저 확정하고, 수정마다 실물 확인과 전수 검사를 거쳐 배포 후 재확인. 5/5 달성, 기획→배포 2시간 11분.

**Q. 왜 프레임워크를 안 썼나** — 상태(state) 없는 정적 페이지에 프레임워크는 과잉이다. React 경험은 사내 LMS(200+ 화면)로 이미 검증했고, 이 문제의 크기에는 바닐라가 맞는 도구였다. 인터랙티브 레이더까지 바닐라 JS로 충분했다.

**Q. AI가 다 한 것 아닌가** — 타이핑은 AI가 다 했다. 문제 정의, 완료 기준, 무엇을 공개하지 않을지, 그리고 20회의 방향 수정이 사람의 몫이었다. AI Native가 조종과 검증의 능력이라면, 이 리포트가 그 증거다.
